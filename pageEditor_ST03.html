Feedback Page

Technical Description

This page is a feedback form designed using HTML and CSS. The HTML structure employs key semantic tags like <form>, <fieldset>, and <legend> to group related form elements and improve the form's clarity. The <form> tag defines the form's action and method (POST), ensuring secure data submission. The enctype="multipart/form-data" attribute allows file uploads. Inside the form, <fieldset> elements are used to categorize information into sections such as "Personal Details," "Feedback Details," and "Rating," with each section labeled appropriately using <label> tags. The use of input types such as text, email, tel, date, datetime-local, and radio allows for a variety of user interactions. CSS styling is applied to enhance design and functionality, such as adding padding, borders, and background colors for form elements. Additionally, hover effects on the submit button and radio button labels improve user interactivity. The form uses responsive styling to maintain usability across different screen sizes.

Accessibility

Accessibility is a key focus in this feedback form. Key elements like alt text for images ensure that visually impaired users have an understanding of the content through screen readers. The use of <label> tags associated with each form input allows screen readers to correctly announce the labels when users navigate through the form. The for attribute in labels is linked to the respective input fields, improving navigation for users relying on assistive technologies. Additionally, the radio buttons and checkboxes are properly labeled to provide context for their options. Placeholder text is used in text inputs to give users clear guidance on what information is required. Furthermore, the form supports keyboard navigation, allowing users to easily tab through the fields. The form's structure and styling are optimized for users with disabilities, ensuring that the form is both usable and accessible.

Team Page

Technical Description

The page structure is primarily built using HTML elements such as <header>, <main>, and <footer>. These provide the framework for organizing the content and separating sections for the website. The <h1> tag is used to define the main heading, while the <div> containers are utilized to group related content, like the team members. Each team member is placed inside a .team-member container, with the use of position: relative and position: absolute properties ensuring that details (such as name and role) appear at the bottom of the image.

The CSS utilizes Flexbox (display: flex) to create a responsive, grid-like layout for the team members, allowing them to wrap and align dynamically. Each team member’s image is styled with a border, height, and width specifications, while the transform property on hover creates an interactive zoom effect. Additionally, the details container is hidden by default (opacity: 0) and revealed when the .team-member is hovered over or focused (using :hover and :focus selectors), ensuring an engaging and accessible interaction. Transitions are added to provide smooth effects when hovering over images and displaying the details. The tabindex="0" on each .team-member allows keyboard users to focus on team members, making the page more accessible.

Accessibility

Accessibility has been incorporated using several techniques to ensure the page is usable by all users, including those with visual or motor impairments. The alt attribute is applied to images, which is essential for screen readers, providing descriptive text for each team member's photo. The tabindex="0" allows keyboard navigation, enabling users to focus on individual team member sections, especially important for those using keyboard-only navigation or assistive technologies. The details section, hidden by default, is revealed through hover and focus events, ensuring that both mouse and keyboard users can interact with the content. The focus states are managed with the :focus pseudo-class, making the page navigable for users with disabilities. The layout uses a simple structure with proper semantic HTML, ensuring compatibility with screen readers and improving the overall accessibility of the page.

Content Page

Technical Description

The HTML structure of the page is designed using semantic elements like <header>, <nav>, <main>, <section>, and <footer>, which help organize content and improve the page's accessibility and SEO. The <header> contains the main title and navigation menu, while the <nav> tag provides links to different sections of the site, ensuring easy navigation. The content is divided into sections using <section>, each representing a topic like "Forests" and "Wildlife." These sections include relevant images and text to support the content. Key CSS properties, such as font-size, color, and margin, are used to style the page. The external CSS files (template_style.css and content_style.css) handle overall page layout, colors, typography, and responsiveness, ensuring a user-friendly design across different devices. The id attributes are used for linking navigation and "go to top" functionality.

Accessibility

Accessibility is prioritized by using proper HTML elements to ensure the content is accessible to all users. The <img> tags have descriptive alt attributes, providing text alternatives for screen readers. Semantic HTML tags like <header>, <main>, <section>, and <nav> help organize the page structure, making it easier for assistive technologies to understand the content. Keyboard navigation is supported through anchor links (<a>) for both top-of-page and section navigation. The use of appropriate heading tags (<h3>, <h2>, <h1>) maintains a logical content structure, which benefits screen readers and users navigating via keyboard. The Go to Top link allows users to easily navigate back to the header. These considerations help ensure the page is accessible to individuals with disabilities.

Challenges and Lessons Learned


  • Challanges :

    1. Creating the website in a visually apealing layout.
      • Used semantic HTML (nav, main, header, footer)
    2. Ensuring the page layout adapts well across different screen sizes.
      • Used flexbox for the team section and relative units (%, vh, vw) for better responsiveness.
    3. Making the navigation menu intuitive and ensuring easy access to different sections.
      • Used a structured navigation bar with labeled links.
      • Added internal linking within content sections for better user experience.
    4. Making the feedback form user-friendly while maintaining accessibility
      • Used proper label elements linked to form controls for better screen reader support.
      • Included placeholder text and title attributes to guide users.
      • Ensured sufficient contrast for readability and hover effects for better visibility.
    5. Creating smooth, interactive effects without using JavaScript.
      • Implemented CSS-only transitions for hover effects, ensuring a dynamic experience without compromising security.


  • Lessons Learned :

    • Proper use of semantic HTML improves accessibility and usability.
    • Flexbox simplifies responsive design and layout consistency.
    • Well-structured navigation enhances the overall usability of the website.
    • CSS can effectively replace JavaScript for basic interactivity.
